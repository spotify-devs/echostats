title: Endpoints
description: Complete API endpoint reference for EchoStats.
---


All endpoints below are relative to the base URL `http://localhost:8000`. Versioned endpoints require JWT authentication via the `session` cookie unless noted otherwise.

---

## Health

Health and readiness probes. **No authentication required.**

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/health` | Health check with service name and version |
| `GET` | `/api/health/ready` | Readiness probe — verifies MongoDB and Redis connectivity |
| `GET` | `/api/health/update` | Checks GitHub for newer releases |

### `GET /api/health`

```json
{
  "status": "healthy",
  "service": "echostats-api",
  "version": "0.1.0"
}
```

### `GET /api/health/update`

```json
{
  "update_available": true,
  "current_version": "0.1.0",
  "latest_version": "0.2.0",
  "release_url": "https://github.com/spotify-devs/echostats/releases/tag/v0.2.0"
}
```

## Authentication

OAuth 2.0 flow with Spotify and JWT session management.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/auth/login` | Generate Spotify authorization URL and redirect |
| `GET` | `/api/v1/auth/callback` | Spotify OAuth callback — exchanges code for tokens, sets session cookie |
| `POST` | `/api/v1/auth/refresh` | Refresh the JWT session token |
| `GET` | `/api/v1/auth/status` | Get current authentication status and user info |
| `POST` | `/api/v1/auth/logout` | Clear session cookie and log out |
| `GET` | `/api/v1/auth/dev-login` | Development-only: bypass OAuth for local testing |

### `GET /api/v1/auth/status`

```json
{
  "authenticated": true,
  "user": {
    "spotify_id": "user123",
    "display_name": "John",
    "email": "john@example.com",
    "avatar_url": "https://i.scdn.co/image/..."
  }
}
```

## Tracks

Track analytics and details.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/tracks/top` | Top tracks ranked by play count |
| `GET` | `/api/v1/tracks/{track_id}` | Detailed track info including audio features |

### `GET /api/v1/tracks/top`

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `period` | string | `month` | Time range: `week`, `month`, `quarter`, `year`, `all_time` |
| `limit` | int | `20` | Number of results (max: 100) |
| `page` | int | `1` | Page number |

**Response:**

```json
{
  "tracks": [
    {
      "track_id": "spotify:track:abc123",
      "title": "Anti-Hero",

        "danceability": 0.637,
        "energy": 0.535,
        "valence": 0.533
      }
    }
  ],
  "total": 312,
  "page": 1,
  "limit": 20
}
```

## Listening History

Raw listening history with pagination and import.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/history` | Paginated listening history |
| `POST` | `/api/v1/history/import` | Import Spotify extended streaming history (JSON) |

### `GET /api/v1/history`

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `page` | int | `1` | Page number |
| `limit` | int | `20` | Results per page (max: 100) |

**Response:**

```json
{
  "history": [
    {
      "track_name": "Blinding Lights",
      "artist_name": "The Weeknd",
      "album_name": "After Hours",
      "played_at": "2024-01-15T14:32:00Z",
      "duration_ms": 200040,
      "image_url": "https://i.scdn.co/image/..."
    }
  ],
  "total": 15842,
  "page": 1,
  "limit": 20
}
```

### `POST /api/v1/history/import`

Upload Spotify extended streaming history JSON files (from [Spotify Privacy Data Export](https://www.spotify.com/account/privacy/)).

**Content-Type:** `multipart/form-data`

## Player

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/player/current` | Currently playing track |
| `GET` | `/api/v1/player/devices` | Available playback devices |

| `POST` | `/api/v1/player/play` | Start or resume playback |
| `POST` | `/api/v1/player/pause` | Pause playback |
| `POST` | `/api/v1/player/next` | Skip to next track |
| `POST` | `/api/v1/player/previous` | Go to previous track |

## Sync Jobs

Background data sync management.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/sync-jobs` | List sync jobs with status and timing |
| `GET` | `/api/v1/sync-jobs/stats` | Sync job statistics (success rate, avg duration) |
| `POST` | `/api/v1/sync-jobs/trigger` | Manually trigger a data sync |

### `GET /api/v1/sync-jobs`

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `page` | int | `1` | Page number |
| `limit` | int | `20` | Results per page (max: 100) |
| `status` | string | — | Filter by status: `pending`, `running`, `completed`, `failed` |
| `job_type` | string | — | Filter by job type |

**Response:**

```json
{
  "jobs": [
    {
      "id": "abc123",
      "job_type": "spotify_sync",
      "status": "completed",
      "started_at": "2024-01-15T14:00:00Z",
      "completed_at": "2024-01-15T14:00:12Z",
      "duration_seconds": 12,
      "items_synced": 47,
      "api_calls": 8
    }
  ],
  "total": 284,
  "page": 1,
  "limit": 20
}
```

### `POST /api/v1/sync-jobs/trigger`

Triggers an immediate sync outside the normal 15-minute schedule.

```json
{
  "job_id": "def456",
  "status": "pending",
  "message": "Sync job queued"
}
```
